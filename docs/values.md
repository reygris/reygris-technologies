# Core Values

ReyGris values are not posters on a wall. They are filters for hiring, partnering, designing, and shipping.

## 1. Vigilance

We notice what others overlook — edge cases, regressions, unclear requirements, and risks before they become incidents.

**In practice:**
- Review work as if it will run in production tomorrow
- Ask "what could go wrong?" early, not after launch
- Document decisions so context is never lost

## 2. Rootedness

We honor where we come from. Culture, language, and personal history inform how we build — they are strengths, not things to hide.

**In practice:**
- Use authentic voice in client and public communication
- Design for real communities, not abstract "users"
- Celebrate origin without excluding anyone

## 3. Craft

Good enough is not our default. We take pride in clean architecture, readable code, and interfaces that respect people's time.

**In practice:**
- Refactor when clarity demands it
- Prefer maintainable solutions over clever ones
- Test behavior that matters

## 4. Clarity

We communicate directly. No filler, no performative complexity, no hiding behind buzzwords.

**In practice:**
- Write docs a new teammate can follow
- Give honest timelines and tradeoffs
- Say "I don't know" when we don't — then find out

## 5. Stewardship

We protect users, data, teammates, and reputation. Trust is earned in small moments and lost in one careless move.

**In practice:**
- Treat security and privacy as product features
- Leave systems better than we found them
- Own mistakes quickly and fix them thoroughly

## 6. Territory

Like the Gray Kingbird, we take responsibility for our domain. When ReyGris touches a project, quality and accountability come with us.

**In practice:**
- Define scope clearly; deliver fully within it
- Do not ship what we would not run ourselves
- Support what we build through meaningful handoff or ongoing care

## Behavioral anti-patterns

We explicitly reject:

| Anti-pattern | Why |
|--------------|-----|
| Shipping to hit a date without disclosure | Erodes trust |
| Copy-paste solutions without understanding | Creates debt |
| Dismissing accessibility or i18n as "later" | Excludes people |
| Blame over learning | Stops improvement |
